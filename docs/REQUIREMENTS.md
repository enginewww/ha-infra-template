## Требования к запуску

### 1. Системные требования

* **Операционная система:** Linux/macOS/Windows с поддержкой Docker.
* **Процессор:** минимум 2 ядра.
* **ОЗУ:** не менее 2 ГБ (рекомендуется 4 ГБ).
* **Место на диске:** минимум 20 ГБ свободного пространства для образов Docker и контейнеров.
* **Сетевые настройки:** возможность работы с локальными виртуальными сетями Docker.

### 2. Программное обеспечение (версии на которых работает стабильно)

* **Docker:** версия 29.2.0
  Проверка:

  ```bash
  docker --version
  ```
* **Docker Compose:** версия 5.0.2
  Проверка:

  ```bash
  docker compose version
  ```
* **Git:** версия 2.43.0
  Проверка:

  ```bash
  git --version
  ```
* **SSH-клиент:** версия 3.0.1
  Проверка:

  ```bash
  ssh -V
  ```

### 3. Настройка проекта

* Наличие прав на создание Docker контейнеров (обычно через `sudo` или группа `docker`).
* Доступность портов для:

  * HTTP: `80` (VIP на балансировщиках)
  * Grafana: `3000` (используется в тестовом режиме)
  * Victoria Metrics: по умолчанию `8428` (используется в тестовом режиме)
* Возможность генерации SSH-ключей в директории проекта (`./.ssh/id_ed25519`).

### 4. Ограничения и рекомендации

* Не запускать несколько экземпляров проекта на одном порту VIP одновременно.
* Для корректного переключения VIP между балансировщиками необходима работа контейнерной сети Docker без конфликтов IP.
* Проверяйте, что firewall или антивирус не блокируют порты контейнеров.
